The koan artifact-comment routing model (`koan/web/app.py`, `koan/state.py`, `koan/phases/format_step.py`) was redesigned on 2026-04-26 during M5 of the unified-artifact-flow initiative. Leon endorsed via plan-spec / execute (clean execution; 715 tests passed). The change replaces the inline-review apparatus (`/api/artifact-review` endpoint, `koan_artifact_propose` blocking-review semantics, `_render_review_payload` per-block payload renderer, `artifact_review_future` parking primitive on `InteractionState`, `ActiveArtifactReview` projection field on `Run`, `artifact_review_started` / `artifact_review_cleared` events) with a simpler comments-as-steering channel: a new `/api/artifact-comment` POST endpoint at `koan/web/app.py` accepts `{path, comment, attachments}`; if `interactions.yield_future` is active, resolves it with the message; otherwise enqueues a `ChatMessage(content=comment, artifact_path=path, ...)` to the steering queue mirroring api_chat's routing logic. `ChatMessage` (a `@dataclass`, not a Pydantic model -- so the new field doesn't trigger memory entry 107's cross-class validation strictness) gained an optional `artifact_path: str | None = None` field at `koan/state.py`; `format_step.steering_message_block()` and `format_steering_messages()` render `[artifact: {path}]` prefix when the field is set, otherwise the message is indistinguishable from a chat message. Doctrine settled during M5 plan-spec: comments are routing-tagged chat messages, not version-anchored proposals -- the orchestrator always assumes the latest artifact version (no version pinning). The patterns settled here: (a) replace endpoints rather than repurpose (clean break -- `/api/artifact-review` deleted, not redirected to the new route); (b) typed routing-metadata fields (`artifact_path`) rather than magic-prefix in content. The `koan_artifact_propose` blocking semantics were superseded by `koan_artifact_write` (memory entry on the new write tool). M6 frontend integration: `submitArtifactReview` replaced with `submitArtifactComment(path, comment, attachments)` posting to the new endpoint.
