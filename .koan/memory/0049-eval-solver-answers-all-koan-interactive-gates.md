The koan eval runner's interactive-gate-handling strategy (at `evals/runner.py` as of 2026-04-23) was revised three times. On 2026-04-17, Leon originally adopted an SSE-driven design in which the solver subscribed to `/events`, detected `/run/activeYield` and `/run/focus` patch ops, and POSTed a fixed "Please use your best judgment" message to `/api/chat` and `/api/answer` to unblock every gate. That design was retired because it duplicated gate-detection logic the koan server already performs internally and did not extend cleanly to per-phase scoring.

On 2026-04-19, Leon adopted a two-part replacement in `evals/solver.py`: (1) the solver flipped `app_state.yolo = True` on its in-process `AppState` handle before calling `create_app()`, so koan's existing yolo-mode auto-response paths (`_yolo_yield_response` and `_yolo_ask_answer` in `koan/web/mcp_endpoint.py`) resolved every `koan_yield` and `koan_ask_question` with the recommended suggestion or option (falling back to "use your best judgement") without solver involvement; (2) after `projection.run.completion` is non-null, the solver harvested per-phase data from `ProjectionStore.events` -- phase-bucketed `koan_ask_question` / `koan_yield` / `koan_set_phase` / `koan_request_scouts` / `koan_memorize` / `koan_search` tool_called events plus `artifact_*` events walked against `phase_started` boundaries -- for downstream per-phase scoring.

On 2026-04-22 during the Inspect-to-DeepEval migration, Leon renamed `evals/solver.py` to `evals/runner.py`, removed the Inspect `@solver` decorator and the `solve(state: TaskState, generate) -> TaskState` signature, and replaced them with a plain `async def run_koan(case: Case) -> dict` whose sole output is the harvest dict. The `transcript().info(...)` progress lines were replaced by `logging.info(...)` on the `koan.evals.runner` logger.

On 2026-04-23 during DeepEval integration debugging, Leon reversed the in-process design: `evals/runner.py` now spawns koan as a subprocess via `python -m koan run --yolo --directed-phases ...` rooted at the fixture submodule, rather than calling `create_app()` in the eval process. Rationale captured in the companion lesson entry: in-process reuse created a version mismatch whenever the fixture SHA differed from the eval harness SHA, because the orchestrator CLI read fixture-snapshot source while the server ran current-tree code, breaking phase transitions. The yolo auto-response mechanism, directed-phase sequencing, and post-hoc projection harvest are unchanged; only the process boundary moved. Harvest now reads the subprocess's projection via the koan HTTP API (`/events` SSE snapshot + `/api/run`) rather than an in-process `ProjectionStore` handle. The surrogate-user-LLM alternative remains rejected for the same reason as before (added cost, non-determinism).

On 2026-04-26 during M7 of the unified-artifact-flow initiative, Leon directed the removal of summary-based scoring from the eval framework after curation surfaced that M5's deletion of `phase_summaries` from `koan/projections.py:Run` had broken the active eval harness (M5's plan-spec analysis missed the harness's `evals/harvest.py:29` and `evals/scorers.py:243, 291` dependencies on the field). M7 dropped the `phase_summaries` extraction in `harvest_run`, deleted `_payload_summary`, rewrote `_payload_overall` and `_payload_workflow` to drop the `## summary` blocks (with `_payload_workflow` switching its phase-ordering fallback from `summaries.keys()` to `artifacts_by_phase.keys()`), removed 8 summary rubric rows from `evals/rubrics.py` (2 fixture-level + 6 task-addendum), surgically edited 3 cross-phase rubric bodies (removed the "Phase summaries form a coherent chain" criterion line; revised the count from "all four" to "all three"), and shrank the `SECTIONS` tuple in `tests/evals/test_koan.py` to `("questions", "artifacts", "overall")`. Eval scoring now relies entirely on artifact-based payloads (the per-phase `questions`, `artifacts`, `overall` sections + the workflow-level `_payload_workflow`) plus programmatic metrics (`DurationMetric`, `TokenCostMetric`, `ToolCallCountMetric`) plus the cross-phase coherence rubric (with the summary-chain criterion gone). Net change: 6 files modified, +17 / -94 LOC (deletion-heavy). The eval-fixture pinned snapshot at `evals/fixtures/koan-1/repo/` retains its original code per memory entry 93. Test count baseline preserved at 715 for `pytest --ignore=tests/evals` (M7 doesn't touch koan core); `deepeval test run tests/evals/test_koan.py` is the API-key-gated full-eval verification.
