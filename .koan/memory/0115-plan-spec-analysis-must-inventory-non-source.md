This entry records a defect in koan's M5 plan-spec analysis under the unified-artifact-flow initiative. On 2026-04-26 during M5, Leon's plan-spec for the inline-review backend removal deliberately scoped the change to delete `phase_summaries` from `koan/projections.py:Run`, the corresponding fold case, the `phase_summary_captured` event, and the capture block in `koan_yield`. The plan-spec's "Files NOT modified" list correctly named the eval-fixture pinned snapshot (`evals/fixtures/koan-1/repo/`, out of scope per memory entry 93) and the compiled frontend bundles (`koan/web/static/app/assets/`, M6 rebuilds) but did NOT enumerate the active eval-harness modules (`evals/harvest.py`, `evals/scorers.py`, `evals/rubrics.py`, `evals/runner.py`, `tests/evals/conftest.py`, `tests/evals/test_koan.py`) which import koan types and consume the `phase_summaries` field directly via `store.projection.run.phase_summaries`. The defect surfaced post-M5 only because Leon explicitly asked the orchestrator during the curation step: "investigate whether or not we accounted for the eval framework's use of summaries when we removed them" -- without that steering, the dogfooded run would have shipped M5 with broken eval harvest (`AttributeError` on the deleted field at `evals/harvest.py:29`) and degraded scoring (placeholder text "(no summary captured for this phase)" returned for all per-phase summary-based scoring at `evals/scorers.py:243, 291`). Root cause: the plan-spec analysis enumerated source-tree consumers (phase prompts, system prompts, permission fence) but did not separately enumerate non-source consumers (eval harness, scripts, integration tests, doc snippets). M7 was added as a follow-up milestone on 2026-04-26 to remove summary-based scoring from the eval framework entirely (memory entry 49 updated). Lesson for future field-removal milestones in koan: when removing a field from `koan/projections.py`, `koan/state.py`, `koan/events.py`, or any koan-core type, plan-spec must inventory non-source consumers in addition to source-tree consumers. The audit checklist: (a) `evals/` -- eval harness modules import koan types and read projection state directly; (b) `scripts/` -- maintenance scripts that may parse koan state files; (c) integration tests under `tests/evals/`, `tests/integration/`, etc.; (d) doc snippets in `docs/` that exemplify the field's shape; (e) any `*.py` that imports from `koan.projections`, `koan.state`, or `koan.events`. The "Files NOT modified" list in plan-spec must be exhaustive enough to cover all dependents.
