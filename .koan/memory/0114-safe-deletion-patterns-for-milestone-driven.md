The koan codebase accumulated several safe-deletion patterns during M5 and M6 of the unified-artifact-flow initiative on 2026-04-26 when retiring `koan_artifact_propose`, `_render_review_payload`, `artifact_review_future`, `phase_summaries`, `format_phase_complete`, and the inline-review frontend molecules (`ArtifactReviewPin`, `ReviewBlock`, `ReviewComment`, `ReviewCommentInput`, `ReviewEvent`). Leon's plan-spec for M5 codified the patterns; execution validated them (M5: 715 tests passed with zero regressions; M6: `npx tsc --noEmit` clean, 715 baseline preserved). The patterns: (1) **Migrate callers BEFORE deleting tools**: M5's plan steps 1-3 migrated phase prompts (`milestone_spec.py`, `plan_spec.py`), the orchestrator system prompt (`koan/prompts/orchestrator.py`), and the permission fence (`koan/lib/permissions.py`) from `koan_artifact_propose` to `koan_artifact_write`; step 4 then deleted the tool. Tests stayed green throughout because no caller was orphaned. (2) **Total-deletion in one change for coupled mechanisms**: M5 removed the `phase_summaries` field on `Run` + the fold case + the `phase_summary_captured` event + the capture block in `koan_yield` + the third source in `_compose_rag_anchor` + the M2 chat-synthesis section in `intake.py` step 3 ALL in one milestone, avoiding dead prose / dead code in any gap. (3) **Negative-presence assertions in tests**: pattern from M3's `test_format_phase_complete_removed`; M5/M6 added analogous tests (`koan_artifact_propose` not importable from `koan.web.mcp_endpoint`; `phase_summaries` not in `Run.model_fields`; deleted molecule symbols not present in `frontend/src/`). (4) **"Why" comments at deletion sites mark previously-stable contracts**: the residual comment at `koan/state.py:97` documenting where `artifact_review_future` lived; frontend "removed in M6" comments at deletion sites in `App.tsx` and `api/client.ts`. Extends memory entry 37 doctrine (executor rationale comments at implementation sites) to cover deletion sites. (5) **Replace endpoints rather than repurpose**: M5's `/api/artifact-review` -> `/api/artifact-comment` was a clean break (new name reflects new semantics), not a same-route-repurpose. Recommended for future milestone-driven endpoint retirements.
