This entry records the invocation-mode dependency of `@deepeval.log_hyperparameters` against koan's eval harness in `tests/evals/test_koan.py`. On 2026-04-23, Leon diagnosed a wiring failure: `@deepeval.log_hyperparameters`-decorated `hyperparameters()` function in `tests/evals/conftest.py` returning the triple `{orchestrator_model, judge_model, koan_git_sha}` produced the warning "No hyperparameters logged" in the `deepeval test run` output. Initial root cause attribution: `log_hyperparameters(func)` at `deepeval/test_run/hyperparameters.py:61-79` executes `func()` at decoration (import) time, fetches `global_test_run_manager.get_test_run()`, and writes `test_run.hyperparameters = processed_hyperparameters(func())`. Because the conftest module seemed to load before `evaluate()` created its active test run, the assignment appeared to land on an ephemeral run object. Initial fix on 2026-04-23: drop `@log_hyperparameters` from conftest, pass `hyperparameters=HYPERPARAMETERS` kwarg to `evaluate()` instead.

On 2026-04-24, during plan-review for the DAGMetric refactor, Leon re-examined the root cause and identified that the failure mode applies only to plain `pytest` invocation, not to `deepeval test run` CLI. Under `deepeval test run`, the pytest plugin at `deepeval/plugins/plugin.py:20-28` calls `global_test_run_manager.create_test_run(...)` during `pytest_sessionstart`, gated on `get_is_running_deepeval()` which reads the `DEEPEVAL` environment variable set exclusively by the CLI at `deepeval/cli/test.py:174` before invoking `pytest.main`. Because `pytest_sessionstart` fires BEFORE test-module import, the shared test_run already exists when `@deepeval.log_hyperparameters` executes at module scope; the decorator's internal `get_test_run()` call returns that shared run and attaches hyperparameters correctly. `wrap_up_test_run(run_duration, ...)` at `deepeval/cli/test.py:174` then finalizes the run and uploads to Confident AI after pytest exits. Under plain pytest, `pytest_sessionstart` does not fire the test_run creation path; the decorator's `get_test_run()` call triggers a lazy ephemeral creation that is never wrapped up. Leon's resolution on 2026-04-24: mandate `deepeval test run tests/evals/test_koan.py` as the canonical invocation and rely on module-level `@deepeval.log_hyperparameters` for hyperparameter wiring (memory 75 fourth revision). The `evaluate(hyperparameters=)` workaround from 2026-04-23 is no longer needed in this harness. Lesson for future DeepEval work in this repo: the `@deepeval.log_hyperparameters` decorator is CLI-invocation-dependent; it works under `deepeval test run` and fails silently under plain `pytest`. When in doubt about which path is active, inspect the `DEEPEVAL` env var or use the `hyperparameters=` kwarg on `evaluate()` as an invocation-agnostic fallback.
