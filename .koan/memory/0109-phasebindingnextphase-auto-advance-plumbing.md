The koan workflow control flow (`koan/lib/workflows.py`, `koan/phases/format_step.py`, `koan/web/mcp_endpoint.py`) shifted from phase-end-trampoline yields to per-binding auto-advance on 2026-04-26 during M3 of the unified-artifact-flow initiative. Leon endorsed the design through plan-spec / execute / exec-review for M3 (clean execution, 725 tests passed). The change: `PhaseBinding` (frozen dataclass) gained an optional `next_phase: str | None = None` field; bindings populated across both `PLAN_WORKFLOW` and `MILESTONES_WORKFLOW` (intake -> plan-spec / milestone-spec, plan-spec -> plan-review, execute -> exec-review for both; review phases plan-review / milestone-review / exec-review and curation get None). `PhaseContext` gained `next_phase: str | None` and `suggested_phases: list[str]` fields populated at the `_step_phase_handshake` site from `workflow.get_binding(current_phase).next_phase` and `get_suggested_phases(workflow, current_phase)`. A pure helper `terminal_invoke(next_phase, suggested_phases) -> str` in `koan/phases/format_step.py` (no app_state or workflow imports; no dependency cycles) renders the last-step `invoke_after` directive: `koan_set_phase("X")` (auto-advance) when bound, `koan_yield(suggestions=[...])` (full yield) when None. The text includes an explicit override clause: "If exceptional circumstances warrant user direction, call koan_yield instead." The `format_phase_complete` trampoline at `koan/phases/format_step.py:109` was deleted; its call site at `koan/web/mcp_endpoint.py:986` (the `next_step is None` branch in `koan_complete_step`) was replaced with a defensive fallback message nudging the LLM back to the right path if it accidentally calls `koan_complete_step` at the boundary. All eight phase modules' last-step `step_guidance` now return `StepGuidance(..., invoke_after=terminal_invoke(ctx.next_phase, ctx.suggested_phases))`. Auto-advance is GUIDANCE not enforcement -- the orchestrator may yield instead when warranted; documented in the new `docs/guided-transitions.md` (121 lines). Yolo / `directed_phases` short-circuit (memory entry 61) is independent and continues to work in eval mode. Eval-fixture pinned snapshot at `evals/fixtures/koan-1/repo/` retains the old symbol intentionally per memory entry 93.
